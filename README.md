# Excel-Dr

Excel-Dr，中文可以理解为“Excel 医生”。

它不是一个笼统的“表格优化器”，而是一把更像手术刀的工具：专门针对那些因为历史复制、隐藏对象残留、异常绘图层膨胀而变慢、变卡、打开关闭都痛苦的 Excel / WPS 报表，做**精准识别**与**精准清理**。

很多财务、运营、客服、仓储、订单、行政团队每天都在维护大型报表。真正拖慢表格的，往往不是你肉眼看到的几千行数据，而是埋在 `.xlsx` 内部结构里的垃圾对象、损坏关系、异常隐藏绘图数据。这类问题很难靠手工排查，但 Excel-Dr 可以。

## 它能解决什么问题

- 打开报表时出现“正在加载大量对象”的提示
- 关闭、保存、筛选、滚动明显卡顿
- 明细行数不算很大，但文件体验异常糟糕
- 从旧报表复制表头或模板后，新表也继承了卡顿问题
- WPS / Excel 文件里积累了看不见的隐藏垃圾对象

## 核心价值

- **精准**：重点识别异常膨胀的隐藏 `drawing` 对象，而不是粗暴删除一切图片或对象
- **安全**：默认不改动单元格公式、共享字符串、样式、`cellImages` 图片公式
- **轻量**：Windows 便携版，解压即用，删目录即卸载
- **实用**：面向真实办公场景，不需要懂 XML、不需要懂 Excel 内部结构
- **高收益**：对于“打开慢、关闭慢、保存慢”的报表，往往能明显改善体验

## 适用场景

- 财务月度订单汇总表
- 电商发货清单、售后台账、对账表
- 微信群 / 人工录入 / 多人接力维护的 Excel 报表
- 运营部门长期沿用的历史模板
- 复制来复制去、格式复杂、越来越卡的 WPS 表格

## Excel-Dr 的识别策略

Excel-Dr 不是“见对象就删”，而是做保守判断，只清理明显像垃圾残留的内容。

当前策略主要包括：

- 只处理 `.xlsx`
- 重点检查 `drawing` 绘图层
- 只关注**隐藏对象**
- 只有当同一隐藏图片资源在同一锚点或极少数锚点上**大量重复出现**时，才判定为异常膨胀
- 同时会清理明显损坏的 `#REF!` 数据有效性规则

这意味着：

- 正常可见图片默认不会删
- 正常图表、普通对象默认不会删
- 单元格公式不会改
- 样式和文本不会改
- `cellImages.xml` 里的单元格图片公式不会动

## 为什么这个工具有意义

现实中的 Excel 卡顿问题，很多时候不是“电脑不行”，也不是“数据行数太多”，而是报表在长期复制、粘贴、继承模板、多人接力编辑的过程中，悄悄积累了大量隐藏垃圾数据。

这类问题有三个典型特征：

1. 业务人员感觉“表越来越卡”，但找不到原因
2. 复制一部分内容到新表后，卡顿还会继承
3. 手工清理几乎无从下手

Excel-Dr 的价值，就在于把这种“难定位、难解释、难清理”的问题，变成一个普通用户也能操作的流程：

1. 先检测
2. 看报告
3. 确认后清理
4. 输出新文件

## 主要功能

### 1. 单文件检测

快速扫描一个 `.xlsx` 文件，输出：

- 是否存在异常隐藏对象
- 预计清理数量
- 是否存在损坏数据有效性
- 每张工作表的大致命中情况

### 2. 单文件清理

对异常文件进行保守清理，并另存为新文件：

- 不覆盖原文件
- 自动生成 `_cleaned.xlsx`
- 清理前有确认弹窗

### 3. 文件夹批量检测

扫描整个文件夹内的 `.xlsx` 文件，汇总输出：

- 扫描文件数
- 需要处理的文件数
- 异常隐藏对象总量
- 正常文件与异常文件列表

### 4. 文件夹批量清理

只处理真正命中异常的文件：

- 正常文件自动跳过
- 每个异常文件单独输出一个 `_cleaned.xlsx`
- 清理前先给出简洁确认提示

## 相比手工排查的优势

- 不需要解压 `.xlsx` 手工删 XML
- 不需要懂 Excel 对象关系
- 不需要在 WPS / Excel 里一个个试
- 不会因为误操作直接破坏原文件
- 比“全量另存、盲目复制、重做模板”更省时间

## 运行方式

### 便携版

推荐直接使用 Windows 便携版：

- 解压
- 双击 `Excel-Dr.exe`
- 用完直接删掉目录即可

### Python 运行

```powershell
python cleaner.py
```

### 命令行

单文件：

```powershell
python cleaner.py --scan "C:\path\file.xlsx"
python cleaner.py --clean "C:\path\file.xlsx" --output "C:\path\file_cleaned.xlsx"
```

文件夹：

```powershell
python cleaner.py --scan-folder "C:\path\folder"
python cleaner.py --clean-folder "C:\path\folder"
```

## 打包

```powershell
build_exe.bat
```

或手动：

```powershell
pip install pyinstaller
pyinstaller --noconsole --onefile cleaner.py --name Excel-Dr
```

- 贡献更强的检测策略

如果你遇到某些报表很卡，但 Excel-Dr 没命中，或者命中后你希望它识别得更细，可以提交一个可复现样本或结构特征描述。

## 安全提醒

- 工具默认另存为新文件，不覆盖原文件
- 第一次处理重要报表时，仍建议先用副本测试
- 如果你的文件包含特殊插件、宏、第三方扩展对象，建议先做一份备份

## 项目愿景

Excel-Dr 想做的，不只是“清理几个对象”。

它要成为一个让普通办公用户也能真正用起来的 Excel 报表修复工具：能解释问题、能定位问题、能处理问题，而且足够轻、足够稳、足够直接。

让那些本来应该服务工作的报表，不再反过来拖累工作。
