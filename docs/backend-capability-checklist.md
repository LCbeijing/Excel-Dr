# Excel-Dr 后端能力与验收清单

## 第一阶段能力边界

第一阶段 Rust 后端必须完整替代当前 Python 版本，不是单文件 MVP。第一阶段只限制“问题类型”不盲目扩张，但现有流程能力必须完整重构，包括单文件、文件夹批量、进度、报告、错误处理和 Windows 打包。

| 能力 | 检测 | 修复 | 第一版是否支持 | 说明 |
| --- | --- | --- | --- | --- |
| 异常隐藏 drawing 对象 | 支持 | 支持 | 是 | 当前 Python 已有规则，可迁移到 Rust |
| 损坏数据有效性规则 | 支持 | 支持 | 是 | 重点处理明显 `#REF!` 规则 |
| 文件夹批量检测 | 支持 | 不涉及 | 是 | 先扫描，再决定哪些文件可清理 |
| 文件夹批量清理 | 支持 | 支持 | 是 | 只处理命中文件，正常文件跳过 |
| 普通图片压缩 | 不支持 | 不支持 | 否 | 风险高，容易破坏用户内容 |
| 样式膨胀清理 | 可调研 | 暂不支持 | 否 | 需要样本和规则验证 |
| 共享字符串清理 | 可调研 | 暂不支持 | 否 | 误伤风险较高 |
| 外部链接修复 | 可检测 | 暂不支持 | 否 | 第一版只提示，不自动改 |

## 问题类型设计

### 1. 异常隐藏 drawing 对象

用户价值：

- 解决打开慢、保存慢、筛选卡顿、文件明显变重的问题。

检测输入：

- `xl/worksheets/*.xml`
- `xl/worksheets/_rels/*.rels`
- `xl/drawings/*.xml`
- `xl/drawings/_rels/*.rels`

检测策略：

- 只统计隐藏对象。
- 识别重复锚点、重复图片关系、异常数量。
- 达到阈值才判定为可清理。

修复动作：

- 移除命中的隐藏 anchor。
- 清理不再被引用的 drawing relationship。
- 清理不再被引用的 media 文件。
- drawing 为空时移除 worksheet 中对应 drawing 关系。

禁止动作：

- 不删除可见图片。
- 不删除图表。
- 不修改公式。
- 不修改样式。
- 不修改 sharedStrings。
- 不修改 cellImages。

验收：

- 正常图片样本不命中。
- 异常隐藏对象样本命中。
- 输出文件可打开。
- 原文件 hash 不变。

### 2. 损坏数据有效性规则

用户价值：

- 解决模板复制、单元格引用断裂后造成的异常提示、卡顿或不可用规则。

检测输入：

- `xl/worksheets/*.xml`

检测策略：

- 读取 `dataValidations/dataValidation`。
- 检测公式属性或 `formula1` 节点中是否包含明显损坏引用。

第一版规则：

- `#REF!`

修复动作：

- 删除损坏的 `dataValidation` 节点。
- 更新 `dataValidations count`。
- 如果没有剩余规则，移除空的 `dataValidations` 容器。

禁止动作：

- 不修改正常下拉选项。
- 不修改单元格值。
- 不修改公式。

验收：

- 正常数据有效性样本不命中。
- `#REF!` 样本命中并可清理。
- 输出文件可打开。

## Rust 错误类型

后端不要只返回字符串错误，至少需要这些错误码：

| 错误码 | 用户提示 |
| --- | --- |
| UnsupportedFormat | 当前版本只支持 .xlsx 文件 |
| FileNotFound | 找不到选择的文件 |
| PermissionDenied | 没有权限读取或写入该文件 |
| PasswordProtected | 文件可能已加密或受保护，暂不支持处理 |
| InvalidZip | 文件结构损坏，无法读取 |
| MissingWorkbook | 文件缺少工作簿结构，无法处理 |
| XmlParseFailed | 文件内部结构异常，无法安全处理 |
| OutputExistsLocked | 输出文件正在被占用，请关闭后重试 |
| Cancelled | 已取消处理 |
| InternalError | 处理失败，请查看运行日志 |

## 任务状态机

```text
Idle
  -> FileSelected
  -> FolderSelected
  -> Analyzing
  -> AnalyzeCompletedNoIssue
  -> AnalyzeCompletedHasIssue
  -> Cleaning
  -> CleanCompleted
  -> Failed
  -> Cancelled
```

前端按钮状态：

| 状态 | 仅检测 | 清理并新建 |
| --- | --- | --- |
| 未选择文件 | 禁用 | 禁用 |
| 已选择文件 | 可用 | 禁用 |
| 已选择文件夹 | 可用 | 禁用 |
| 检测中 | 禁用 | 禁用 |
| 检测完成，无问题 | 可用 | 禁用 |
| 检测完成，有问题 | 可用 | 可用 |
| 清理中 | 禁用 | 禁用 |
| 清理完成 | 可用 | 可用 |
| 失败 | 可用 | 视报告决定 |

## 输出文件规则

单文件：

```text
原文件：月度订单汇总.xlsx
输出：月度订单汇总_cleaned.xlsx
```

如果输出文件已存在：

- 默认追加序号：`月度订单汇总_cleaned_2.xlsx`
- 不静默覆盖。

文件夹：

- 正常文件不生成副本。
- 命中文件生成对应 `_cleaned` 副本。
- 最后生成批量处理报告。
- 单个文件失败不能中断整批任务。
- 批量结果必须区分已清理、已跳过、失败三类。

## 日志与报告

运行日志用于定位问题，报告用于给用户看，两者必须分开。

用户报告应包含：

- 文件名
- 检测结果
- 命中问题
- 处理动作
- 输出位置
- 跳过文件
- 失败文件及原因

运行日志应包含：

- task_id
- 阶段
- 文件路径
- 错误码
- Rust backtrace 是否开启

日志限制：

- 不记录单元格内容。
- 不上传文件。
- 不默认收集用户隐私路径以外的信息。

## 性能目标

第一版性能目标：

- 50 MB 文件：检测过程界面无假死。
- 100 MB 文件：进度持续更新。
- 5 万个异常 drawing 对象：能完成检测和清理。
- 文件夹 100 个 `.xlsx`：能批量扫描，失败文件不影响其他文件。
- 文件夹批量清理：只处理命中文件，不能重复清理 `_cleaned.xlsx`。

工程目标：

- Rust 后端任务运行在后台线程。
- Flutter 主线程只渲染 UI。
- Rinf 消息只传结构化数据，不传大文件内容。
- 临时文件写入失败时清理现场。

## 开发完成定义

不能只以“能打包”为完成标准，必须同时满足：

- 有测试样本。
- 有自动化测试。
- 有手动验收清单。
- 有 Windows 免安装包。
- 有失败状态截图。
- 有大文件处理截图。
- 有文件夹批量处理截图。
- 有输出文件可打开验证。

## 第二阶段定位

第二阶段目标是把 Excel-Dr 扩展为更完整的 Excel 修复平台，重点增加更多问题类型，而不是补第一阶段遗漏的基础流程。

第二阶段候选能力：

- 无用样式膨胀检测与修复。
- 共享字符串异常膨胀检测。
- usedRange 异常扩张检测。
- 外部链接失效检测。
- 条件格式异常膨胀检测。
- 损坏图片关系检测。
- WPS 兼容性专项检测。
- 更完整的报告导出与问题解释。
